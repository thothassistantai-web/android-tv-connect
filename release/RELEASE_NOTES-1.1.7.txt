Fix HDMI capture audio and Settings dialog freeze (v1.1.7).

- Auto-discover PipeWire/Pulse audio from the MS2109 HDMI capture dongle (matches video auto-discovery).
- Default capture audio is now "auto" instead of a fragile hard-coded ALSA node name.
- When the dongle is connected but settings point at PC built-in audio, the app now uses capture audio instead.
- Existing configs with wrong audio_device are migrated to auto on load.
- Changing the audio device in Settings prompts "Can you hear audio?" with Yes / No — try next cycling.
- Settings Save, Cancel, Escape, and close no longer block on capture pipeline restarts.
- Capture live-apply and GStreamer state changes run off the GTK main thread with short async waits.
